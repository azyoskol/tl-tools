# Research: Features

**Defined:** 2026-05-05
**Sources:** `../docs/STATUS.md`, `../docs/product/roadmap.md`, `../docs/product/onboarding.md`, `../docs/product/personas.md`

## Table Stakes For Community Preview

- Docker Compose install path that starts API, UI, Postgres/TimescaleDB, and Redis.
- First-run demo dataset with a believable organization narrative.
- Overview dashboard with meaningful engineering metrics.
- DORA and core delivery metrics exposed by the backend.
- Basic auth and role-aware access.
- Source connection/onboarding wizard for at least one practical path.
- Clear privacy/self-hosting posture.
- Documentation that accurately says what works.

## Community GA Expectations

- Widget/dashboard builder v1.
- Dashboard templates for major personas: CTO/VP, Engineering Manager, Tech Lead, Developer, DevOps/SRE.
- More source integrations: GitHub/GitLab, Jira/Linear, CI/CD, Prometheus.
- Alerts/notifiers such as email, Slack, or Telegram.
- Exportable data or reports.
- Onboarding progress checklist.

## Pro Differentiators

- License-gated Pro features.
- AI assistant and AI insights grounded in actual metrics.
- Dual-LLM privacy/security architecture.
- Plugin runtime and marketplace.
- Advanced exports and reporting.

## Enterprise Differentiators

- SSO, advanced RBAC, audit logging.
- Air-gapped/on-prem installation hardening.
- Compliance evidence and security posture.
- White-label and regulated-industry readiness.

## Anti-Features

- Individual developer surveillance/scoring.
- Cloud-first architecture that requires customer code/metadata to leave their environment.
- Black-box AI insights without grounding.
- Full ClickHouse dependency in the initial Community Preview runtime.
